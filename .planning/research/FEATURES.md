# Features Research: 브랜드 네이밍 앱

## Table Stakes (필수 — 없으면 쓸모없음)

### 입력 시스템
| 기능 | 복잡도 | 설명 |
|------|--------|------|
| 상품/가게 기본정보 입력 | 낮음 | 업종, 위치, 규모, 상품, 가격대 |
| 사장님 정보 입력 | 낮음 | 비전, 꿈, 목표, 스토리 |
| 브랜드 페르소나 입력 | 중간 | 철학, 슬로건, 미션 등 (기존 앱 항목) |
| 항목별 개별 제출 | 낮음 | 각 항목 입력 후 버튼으로 추천 트리거 |

### 추천 시스템
| 기능 | 복잡도 | 설명 |
|------|--------|------|
| 브랜드명 3개 이하 추천 | 중간 | Gemini API 호출, 프롬프트 엔지니어링 |
| 작명 사유 표시 | 낮음 | 왜 이 이름인지 짧은 설명 |
| 기반 항목 표시 | 낮음 | 어떤 입력을 근거로 추천했는지 |
| 추천 카드 누적 | 낮음 | 우측 패널에 카드가 쌓이는 구조 |
| 누적 컨텍스트 반영 | 중간 | 새 항목 입력 시 이전 입력 전체를 반영한 추천 |

### 지식 기반
| 기능 | 복잡도 | 설명 |
|------|--------|------|
| 문서 업로드 | 중간 | PDF, DOCX 등 브랜딩 자료 업로드 |
| 업로드 자료 기반 추론 | 높음 | RAG — 업로드된 자료만 참조하여 추천 |

### 레이아웃
| 기능 | 복잡도 | 설명 |
|------|--------|------|
| 70/30 분할 레이아웃 | 낮음 | 왼쪽 입력 / 오른쪽 추천 |
| 한 화면 스크롤 | 낮음 | 모든 입력 항목 세로 나열 |

## Differentiators (차별화 — 경쟁 우위)

| 기능 | 복잡도 | 설명 |
|------|--------|------|
| 점진적 정교화 | 중간 | 항목을 채울수록 추천이 정교해지는 경험 |
| 추천 비교/선택 | 낮음 | 마음에 드는 브랜드명 즐겨찾기/선택 |
| 추천 히스토리 타임라인 | 낮음 | 어떤 항목 입력 후 어떤 추천이 나왔는지 흐름 |
| 선택한 브랜드명 내보내기 | 낮음 | 최종 선택 결과를 PDF/텍스트로 내보내기 |
| 세션 저장/불러오기 | 중간 | 작업 중간에 저장하고 나중에 이어서 |
| 프롬프트 투명성 | 낮음 | AI에게 보낸 프롬프트를 볼 수 있는 옵션 |

## Anti-Features (의도적으로 만들지 않을 것)

| 기능 | 이유 |
|------|------|
| 실시간 자동 추천 | 입력 중 계속 API 호출은 비용 낭비 + UX 혼란 |
| 브랜드명 → 페르소나 역방향 | 기존 앱이 담당 |
| SNS/도메인 가용성 체크 | v1 범위 초과, 외부 API 의존 |
| 다국어 브랜드명 | 한국어 전용으로 집중 |
| 협업/공유 기능 | 사내 툴, v1에서는 개인 사용 |

## 기능 간 의존성

```
문서 업로드 → RAG 인덱싱 → 추천 시 참조
기본정보 입력 → 페르소나 입력 → 누적 컨텍스트 → 추천
추천 카드 → 즐겨찾기/선택 → 내보내기
```
