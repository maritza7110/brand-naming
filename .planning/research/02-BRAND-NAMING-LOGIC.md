# 리서치: 브랜드 네이밍 논리적 프레임워크 및 데이터 항목 (v2.0)

## 1. 목적
브랜드 네이밍 결과의 '논리적 타당성'을 확보하기 위해, 전문가 수준의 데이터 수집 항목과 생성 로직을 정의한다.

## 2. 필수 데이터 항목 (Data Schema)

### A. 분석 단계 (Strategic Audit)
- **경쟁사 분석 (Competitor Landscape):** 경쟁 브랜드명, 그들의 의미적 지향점(기능 중심 vs 감성 중심).
- **시장 트렌드 (Market Context):** 현재 산업군의 언어적 트렌드 (예: IT 서비스의 경우 '~ify', '~ly' 등).
- **차별화 요소 (USP):** 우리 제품만이 가진 독보적인 기능이나 가치 1~2가지.

### B. 정체성 단계 (Identity Definition)
- **브랜드 퍼스낼리티 (Personality):** 브랜드가 사람이라면 어떤 성격일지 (친근한, 지적인, 혁신적인 등).
- **타겟 페르소나 (Target Persona):** 핵심 사용자의 인구통계학적 특징 및 라이프스타일.
- **브랜드 비전 (Vision):** 브랜드가 궁극적으로 해결하고자 하는 문제.

### C. 표현 단계 (Verbal Expression)
- **네이밍 스타일 선호도 (Style Preference):**
    - **Descriptive (기술형):** 서비스 내용을 직접 설명 (예: 배달의민족).
    - **Abstract (추상형):** 의미를 은유적으로 전달 (예: Apple).
    - **Compound (합성형):** 단어를 조합 (예: Facebook).
    - **Acronym (두문자형):** 앞 글자를 땀 (예: BMW).
- **언어 제약 (Linguistic Constraints):** 선호 언어 (한글/영어), 글자 수 제한, 발음의 강도.

## 3. 논리적 생성 로직 (Naming Logic 2.0)

1.  **데이터 가중치 부여:** 사용자가 입력한 데이터 중 '가장 강조하고 싶은 키워드'를 선택하게 함 (HITL).
2.  **키워드 확장 (Expansion):** 입력된 핵심 가치로부터 동의어, 유의어, 은유적 키워드를 Gemini가 확장 생성.
3.  **조합 및 필터링 (Synthesis):** 확장된 키워드와 선택된 네이밍 기법을 결합.
4.  **타당성 검증 (Self-Audit):** 생성된 각 후보안에 대해 Gemini가 스스로 '왜 이 이름이 논리적으로 타당한지' 근거를 제시.

## 4. 결론: UI/UX 반영안
- 현재의 단순 입력 폼을 **'단계별 마법사(Wizard)'** 형식으로 변경.
- 각 단계에서 입력된 정보가 어떻게 네이밍에 반영되는지 시각적으로 피드백 제공.
- "이 이름은 [입력한 핵심가치 A]와 [네이밍기법 B]를 결합하여 [타겟 C]에게 [이미지 D]를 전달합니다"라는 논리적 설명 포함.
