# Pitfalls Research: 브랜드 네이밍 앱

## Critical Pitfalls

### 1. RAG 품질 — 한국어 문서 청킹
**문제**: 한국어 텍스트를 영어 기준 청킹 전략으로 나누면 의미 단위가 깨짐
**경고 신호**: 추천 사유가 문서 내용과 동떨어진 답변
**예방 전략**:
- 문단/섹션 단위로 청킹 (문자 수 기반 X)
- 한국어 형태소 분석 불필요 — Gemini Embedding이 한국어 자체 처리
- 청킹 결과를 UI에서 확인할 수 있는 디버그 모드 고려
**관련 페이즈**: RAG 시스템 구축

### 2. 프롬프트 누적 — 컨텍스트 폭발
**문제**: 항목을 10개 이상 입력하면 전체 입력 + RAG 컨텍스트가 토큰 한도 초과
**경고 신호**: API 에러 또는 응답 품질 급격히 저하
**예방 전략**:
- 입력 요약 전략: 이전 입력을 요약본으로 압축
- RAG 검색 결과 수 제한 (top-3~5)
- Gemini 3.1 Pro의 긴 컨텍스트 윈도우 활용 (100만 토큰)
- 그래도 프롬프트 토큰 카운트 모니터링 필수
**관련 페이즈**: AI 서비스 구축

### 3. 브랜드명 품질 — 반복/진부한 이름
**문제**: AI가 비슷비슷한 이름만 반복 추천
**경고 신호**: 추천 카드에 비슷한 패턴의 이름이 반복
**예방 전략**:
- 이전 추천 이름을 프롬프트에 포함하여 "이미 추천한 이름은 제외" 지시
- 다양성 파라미터 (temperature) 조정
- 업로드된 브랜딩 서적의 작명 방법론을 프롬프트에 명시적 활용
**관련 페이즈**: AI 서비스 구축

### 4. API 키 보안
**문제**: 클라이언트 SPA에서 Gemini API 키가 노출됨
**경고 신호**: API 키가 브라우저 개발자 도구에서 보임
**예방 전략**:
- 사내 앱이라 위험도 낮지만, 환경변수로 관리
- 필요시 간단한 프록시 서버 (Vercel Edge Function)로 키 숨김
- API 키에 도메인 제한 설정
**관련 페이즈**: 프로젝트 셋업

### 5. 추천 카드 과다 — UX 혼잡
**문제**: 항목 10개 × 추천 3개 = 카드 30개 → 우측 패널이 카드로 넘침
**경고 신호**: 사용자가 원하는 카드를 찾기 어려워함
**예방 전략**:
- 항목별 그룹핑으로 카드 정리
- 접기/펼치기 기능
- 즐겨찾기로 중요한 추천만 필터링
- 최신 추천이 상단에 오도록 역순 정렬
**관련 페이즈**: UI 구축

### 6. 문서 업로드 — 파싱 실패
**문제**: 스캔 PDF, 이미지 PDF는 텍스트 추출 불가
**경고 신호**: 업로드 후 빈 텍스트 또는 깨진 문자
**예방 전략**:
- 업로드 후 추출된 텍스트 미리보기 제공
- 지원 형식 명확히 안내 (텍스트 기반 PDF, DOCX)
- OCR은 v1 범위 밖 — 명시적으로 제외
**관련 페이즈**: RAG 시스템 구축

## Medium Pitfalls

### 7. Gemini API 응답 형식 불일치
**문제**: AI 응답이 예상 JSON 형식과 다를 때 파싱 실패
**예방**: 구조화된 출력 (JSON mode) 사용 + 폴백 파싱 로직

### 8. 인메모리 벡터 — 브라우저 새로고침 시 소멸
**문제**: 업로드한 문서 임베딩이 새로고침하면 날아감
**예방**: IndexedDB에 임베딩 캐시, 또는 업로드 시 로컬스토리지 활용

### 9. Pretendard 폰트 로딩 지연
**문제**: 웹폰트 로딩 전 FOUT (Flash of Unstyled Text)
**예방**: font-display: swap + 프리로드
