# Requirements: 브랜드 네이밍 앱 v2.0

## 🎯 v2.0 Goals
- 전문가 수준의 브랜드 네이밍 논리 체계 확립 (Naming Logic 2.0).
- 개인별 프로젝트(세션) 관리 및 데이터 자산화.
- 사용자 간 협업 및 영감 공유를 위한 소셜 플랫폼 구축.

## 🛠 Active Requirements (v2.0)

### 1. 지능형 네이밍 로직 (Intelligence)
- [ ] **단계별 입력 프로세스:** '분석-정체성-표현'의 3단계 위저드 UI 도입.
- [ ] **신규 데이터 항목:**
    - 경쟁사 분석, 시장 트렌드, USP(차별화 요소).
    - 브랜드 퍼스낼리티, 상세 타겟 페르소나, 브랜드 비전.
    - 네이밍 스타일(기술/추상/합성/두문자) 및 언어 제약 조건.
- [ ] **논리적 타당성 피드백:** 생성된 각 이름에 대해 '왜 이 이름이 선택되었는지' 입력 데이터와 연결된 근거 제시.
- [ ] **키워드 가중치 설정:** 사용자가 입력한 키워드 중 생성 시 비중을 높일 항목 선택 (HITL).

### 2. 데이터 관리 체계 (Storage & Taxonomy)
- [ ] **네이밍 세션 시스템:** 프로젝트 단위로 입력 데이터와 생성 결과를 묶어서 관리.
- [ ] **아카이빙 기능:** 마음에 드는 결과물을 개인 보관함에 별도 저장 및 메모 추가.
- [ ] **자동/수동 태깅:** 시스템이 분석한 스타일 태그와 사용자가 직접 부여한 커스텀 태그 지원.
- [ ] **분류 필터:** 업종, 스타일, 언어, 평점 기반의 고도화된 검색/필터링.

### 3. 소셜 및 협업 (Social)
- [ ] **인증 시스템:** 이메일/소셜 로그인 및 사용자 프로필 관리.
- [ ] **공개 갤러리 (Public Board):** 사용자가 직접 발행한 네이밍 프로젝트 리스트 제공.
- [ ] **상호작용:** 좋아요, 댓글, 북마크(타인의 이름을 내 보관함으로 가져오기).
- [ ] **인기 랭킹:** 주간/월간 인기 브랜드명 및 트렌드 리더 보드.

### 4. UI/UX 개선 (Experience)
- [ ] **대시보드 UI:** 개인 프로젝트 현황을 한눈에 볼 수 있는 관리 화면.
- [ ] **인터랙티브 가이드:** 네이밍 단계마다 툴팁이나 예시를 통해 전문적인 입력을 유도.
- [ ] **반응형 고도화:** 태블릿 및 모바일 기기에서의 입력 편의성 극대화.
