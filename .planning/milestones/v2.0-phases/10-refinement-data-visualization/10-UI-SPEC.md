## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 아이콘 내부 간격, 배지 내부 패딩 |
| sm | 8px | 칩 사이 gap, 댓글 아이템 사이 gap (`gap-[8px]` — 기존 ChipSelector 패턴) |
| md | 16px | 카드 내부 패딩, 섹션 내부 패딩 (`p-4` = 16px — 기존 GalleryCard 패턴) |
| lg | 24px | 섹션 사이 간격, 필터바 상하 패딩 |
| xl | 32px | 리더보드 섹션 상단 여백, 통계 섹션 상단 여백 |
| 2xl | 48px | 페이지 내 주요 구역 분리 |
| 3xl | 64px | 페이지 상단 여백 (기존 페이지 패턴 유지) |

Exceptions:
- 댓글 입력 폼 textarea: min-height 80px (4의 배수 아니지만 가독성 최솟값)
- 모바일 터치 타겟 (필터 칩 버튼): min-height 36px (44px 아님 — 기존 ChipSelector py-1.5 패턴 유지)
- 리더보드 TOP 배지 순위 번호: 24px × 24px 원형

## Layout & Interaction Contracts

### 1. 갤러리 페이지 레이아웃 (GalleryPage)

```
GalleryPage
├── GalleryLeaderboard      ← 신규: 상단 고정 섹션 (주간/전체 기간 탭)
├── GallerySortTabs         ← 기존: 최신순/인기순 탭
├── GalleryFilterBar        ← 신규: GallerySortTabs 바로 아래 배치
│   ├── [업종 칩 행]
│   ├── [네이밍 스타일 칩 행]
│   └── [키워드 검색 TextField]
└── GalleryGrid             ← 기존: 필터 상태 연동
```

**GalleryLeaderboard 레이아웃:**
- 섹션 헤더: "이번 주 인기 TOP" (20px semibold, text-[#E8E2DA])
- 기간 탭: "이번 주" / "역대" — GallerySortTabs 와 동일한 탭 패턴 (px-4 py-2, border-b)
- 카드 행: 가로 스크롤 `overflow-x-auto` flex 행, 카드 5개 (각 min-w-[200px])
- 카드 구조: 순위 번호 배지(24×24 원형 bg-[#4A4440]) + 브랜드명 + 업종 배지 + LikeButton(showCount)
- 카드 배경: `bg-[#1A1A1E] border border-white/5 rounded-xl p-4`
- 섹션 하단 border: `border-b border-[#4A4440]` (필터바와 구분)

**GalleryFilterBar 레이아웃:**
- 외부 컨테이너: `py-4 border-b border-[#4A4440]`
- 업종 칩 행: ChipSelector 재사용, maxSelection=1 (단일 선택), label="업종"
- 스타일 칩 행: ChipSelector 재사용, maxSelection=1 (단일 선택), label="네이밍 스타일"
- 키워드 검색: TextField 재사용, placeholder="브랜드명 또는 키워드 검색", 앞에 Search 아이콘(lucide)
- 칩 행 간 gap: 16px (md)
- 활성 필터 존재 시 "필터 초기화" 텍스트 버튼 표시 (text-[#A09890] hover:text-[#E8E2DA], 우측 정렬)

**필터 상호작용:**
- 칩 선택 즉시 갤러리 리셋 + 재fetch (디바운스 없음)
- 키워드 검색: 300ms debounce 후 갤러리 리셋 + 재fetch
- 필터 활성 칩: `bg-[rgba(180,140,80,0.15)] text-[#B48C50] border-[#B48C50]` (기존 ChipSelector 패턴)
- 필터 비활성 칩: `bg-[#4A4440] text-[#D0CAC2] border-[#4A4440]` (기존 ChipSelector 패턴)

### 2. GalleryModal 댓글 섹션 (CommentSection)

```
GalleryModal (기존 레이아웃 하단에 추가)
└── <hr border-[#4A4440] my-4>
└── CommentSection
    ├── 섹션 헤더: "댓글 N개" (14px semibold, text-[#A09890])
    ├── 댓글 목록 (max-h-[240px] overflow-y-auto, gap-3)
    │   └── CommentItem × N
    │       ├── 작성자명 (14px semibold, text-[#D0CAC2])
    │       ├── 작성일 (12px, text-[#5A5550]) — date-fns formatDistanceToNow
    │       ├── 본문 (14px, text-[#E8E2DA], whitespace-pre-wrap)
    │       └── [삭제 버튼] — 본인 댓글만 표시, Trash2 아이콘 (lucide, 14px), text-[#5A5550] hover:text-[#EF4444]
    └── 댓글 입력 폼 (인증 사용자만 표시)
        ├── textarea (min-h-[80px], bg-[#1A1A1E], border border-[#4A4440], rounded-lg, p-3, text-[14px], focus:border-[#B48C50], resize-none, maxlength=500)
        └── 제출 버튼: "댓글 작성" (텍스트 버튼, text-[#B48C50] hover:text-[#C5A06B], text-[14px] semibold, 우측 정렬)
```

**GalleryModal 높이 조정:** 댓글 섹션 추가로 모달이 길어지므로 `max-h-[90vh] overflow-y-auto`를 모달 컨테이너에 적용.

**댓글 미인증 상태:** textarea 대신 "댓글을 작성하려면 로그인하세요" (14px, text-[#A09890], text-center py-4).

### 3. 대시보드 통계 섹션 (StatsSection)

```
Dashboard (기존 북마크 탭 아래에 추가)
└── StatsSection (프로젝트 3개 이상 조건부)
    ├── 섹션 헤더: "나의 네이밍 통계" (20px semibold, text-[#E8E2DA], mt-8)
    ├── 차트 그리드: grid-cols-1 sm:grid-cols-3, gap-4
    │   ├── 키워드 빈도 차트 (BarChart, height=180, 가로 수평 막대)
    │   ├── 업종 분포 차트 (PieChart donut, height=220)
    │   └── 네이밍 스타일 차트 (BarChart vertical, height=180)
    └── 데이터 부족 상태 (프로젝트 < 3개): 표시 안 함
```

**차트 공통 스타일 (RESEARCH.md Pattern 3 기준):**
- 배경: `bg-[#1A1A1E] rounded-xl p-4 border border-white/5`
- 차트 타이틀: 14px semibold, text-[#A09890], mb-3
- ResponsiveContainer width="100%"
- Tooltip: `{ backgroundColor: '#1A1A1E', border: '1px solid #4A4440', borderRadius: '8px', color: '#E8E2DA' }`
- 빈 데이터 상태: 14px text-[#5A5550] text-center py-8 "아직 데이터가 없습니다"
