  <name>Task 1: Phase 1 전체 UI 시각 검증</name>
  <files>없음 (검증만 수행)</files>
  <action>
    사용자가 브라우저에서 직접 UI를 확인하는 체크포인트. Claude는 dev server가 실행 중인지 확인하고 (`npm run dev`), 사용자에게 아래 검증 항목을 제시한다.

    검증 대상 (Phase 1 전체 입력 UI):
    - 70/30 분할 레이아웃 (왼쪽 입력 / 오른쪽 추천 대기)
    - 4개 섹션 (매장 기본, 브랜드 정체성, 제품/서비스, 브랜드 페르소나)
    - 18개 입력 필드 (dropdown 3개 + text 1개 + textarea 14개)
    - 프리미엄 디자인 시스템 (Pretendard 폰트, 블루 액센트, 미니멀 여백)

    사용자에게 http://localhost:5173 접속 후 아래 체크리스트 확인 요청:

    **레이아웃:**
    - 왼쪽 70% / 오른쪽 30% 비율이 적절한가
    - 왼쪽 패널이 세로 스크롤되는가
    - 오른쪽 패널이 스크롤해도 고정(스티키)되는가
    - 오른쪽 패널 상단에 backdrop-blur 헤더가 보이는가

    **디자인 퀄리티:**
    - Pretendard 폰트가 적용되어 깨끗하게 렌더링되는가
    - 전반적으로 심플하고 고급스러운 느낌인가 (LAYOUT-04)
    - 여백과 간격이 충분하고 답답하지 않은가
    - 섹션 간 그라데이션 구분선이 미세하게 보이는가

    **입력 필드:**
    - 드롭다운 3개(업종, 규모, 가격대)에서 옵션 선택이 되는가
    - textarea에 텍스트 입력이 되는가
    - 입력 시 포커스 링(파란색 테두리 + 그림자)이 보이는가
    - placeholder 예시문이 회색으로 표시되는가

    **추천 버튼:**
    - 각 섹션 하단에 "추천 받기" 버튼이 있는가
    - 아무것도 입력하지 않으면 버튼이 비활성화(흐리게)인가
    - 1개 항목이라도 입력하면 버튼이 활성화되는가

    **빈 상태:**
    - 오른쪽 패널에 Sparkles 아이콘 + 안내 메시지가 중앙에 표시되는가

    사용자 응답: "approved" 또는 수정이 필요한 부분 구체적 설명
