## Breakpoint Contract

이 Phase의 핵심은 레이아웃 전환이다. 새로운 색상/타이포/컴포넌트가 아니라, 기존 디자인을 모바일에서 사용 가능하게 한다.

| Breakpoint | Range | Layout |
|------------|-------|--------|
| Mobile | < 1024px (default) | 세로 스택: 입력(위) + 추천(아래) |
| Desktop | >= 1024px (`lg:`) | 기존 70/30 좌우 분할 유지 |

**Tailwind 전략:** Mobile-first. 기본값이 모바일, `lg:` 접두사로 데스크톱 스타일 적용.

## Spacing Scale

기존 코드베이스 값 기반. 변경 없이 유지하되, 모바일 예외를 추가한다.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 아이콘 간격, 인라인 패딩 |
| sm | 8px | 컴팩트 요소 간격 |
| md | 16px | 기본 요소 간격 |
| lg | 24px | 섹션 패딩 |
| xl | 32px | 레이아웃 갭 |
| 2xl | 48px | 주요 섹션 구분 |
| 3xl | 64px | 페이지 레벨 여백 |

Exceptions:
- 모바일 터치 타겟 최소 44px (Apple HIG / WCAG 2.5.5)
- 모바일 InputPanel 좌우 패딩: `px-8` (32px) -> `px-5` (20px)
- 모바일 섹션 카드 패딩: `p-7` (28px) -> `p-5` (20px)
- 모바일 히어로 패딩: `px-8 py-9` -> `px-5 py-6`
- 모바일 InputPanel 상단: `pt-12` (48px) -> `pt-6` (24px)
- 모바일 InputPanel 하단: `pb-20` (80px) -> `pb-8` (32px)

## Typography

기존 값 유지. 모바일에서의 크기 조정만 명시.

| Role | Desktop | Mobile | Weight | Line Height |
|------|---------|--------|--------|-------------|
| Body | 14px | 14px (유지) | 400 (regular) | 1.625 (leading-relaxed) |
| Label | 12px | 12px (유지) | 600 (semibold) | 1.5 |
| Heading (Section) | 14px | 14px (유지) | 600 (semibold) | 1.2 |
| Display (Hero) | 26px | 22px | 700 (bold) | 1.2 |
| Sub-label | 11px | 11px (유지) | 600 (semibold) | 1.5 |
| Caption | 10px | 10px (유지) | 700 (bold) | 1.2 |

모바일에서 변경되는 값:
- 히어로 제목 `text-[26px]` -> `text-[22px] lg:text-[26px]`
- 그 외 텍스트 크기는 변경 없음 (14px body는 모바일에서도 적절)

## Layout Transformation Contract

### AppLayout.tsx

**현행 (Desktop):**
```
grid grid-cols-[7fr_3fr] min-h-screen min-w-[1024px]
```

**변경 후 (Responsive):**
```
flex flex-col lg:grid lg:grid-cols-[7fr_3fr] min-h-screen
```

- `min-w-[1024px]` 제거 (최우선 -- STATE.md blocker)
- 모바일: `flex flex-col` -- 입력(위) + 추천(아래) 세로 스택
- 데스크톱: `lg:grid lg:grid-cols-[7fr_3fr]` -- 기존 70/30 유지

### InputPanel.tsx

| Property | Desktop (현행 유지) | Mobile |
|----------|-------------------|--------|
| height | `h-screen` | `h-auto` (자연 높이) |
| overflow | `overflow-y-auto` | `overflow-visible` |
| max-width | `max-w-[640px]` | `max-w-none` (전체 폭) |
| 좌우 패딩 | `px-8` (32px) | `px-5` (20px) |
| 상단 패딩 | `pt-12` (48px) | `pt-6` (24px) |
| 하단 패딩 | `pb-20` (80px) | `pb-8` (32px) |

**Tailwind 구현:**
```
overflow-visible lg:overflow-y-auto h-auto lg:h-screen scroll-smooth bg-[#2C2825]
```
```
max-w-none lg:max-w-[640px] mx-auto px-5 lg:px-8 pt-6 lg:pt-12 pb-8 lg:pb-20
```

### RecommendPanel.tsx

| Property | Desktop (현행 유지) | Mobile |
|----------|-------------------|--------|
| position | `sticky top-0` | `static` (일반 플로우) |
| height | `h-screen` | `h-auto` (자연 높이) |
| overflow | `overflow-y-auto` | `overflow-visible` |
| 최소 높이 | 없음 | `min-h-[300px]` |

**Tailwind 구현:**
```
static lg:sticky lg:top-0 h-auto lg:h-screen overflow-visible lg:overflow-y-auto bg-[#252220] border-t lg:border-t-0 lg:border-l border-[#4A4440]
```

모바일에서 `border-l` -> `border-t`로 구분선 방향 전환.

### Hero Area (InputPanel 내부)

| Property | Desktop (현행 유지) | Mobile |
|----------|-------------------|--------|
| 히어로 패딩 | `px-8 py-9` | `px-5 py-6` |
| 제목 크기 | `text-[26px]` | `text-[22px]` |
| flex 방향 | `flex items-start justify-between` | `flex flex-col gap-4` |
| 버튼 위치 | 제목 옆 우측 상단 | 제목 아래 (D-06) |

**Tailwind 구현 (히어로 컨테이너):**
```
relative rounded-2xl bg-[#363230] px-5 lg:px-8 py-6 lg:py-9 mb-10
```

**Tailwind 구현 (히어로 내부 flex):**
```
relative flex flex-col gap-4 lg:flex-row lg:items-start lg:justify-between
```

**버튼 그룹 (모바일):**
```
flex flex-row gap-2 lg:flex-col lg:items-end lg:gap-2
```

모바일에서 API설정 + 네이밍 초기화 버튼이 가로로 나란히 배치 (D-06).

### Grid Fields (Section Components)

모든 `grid-cols-2` 패턴 -> `grid-cols-1 lg:grid-cols-2` 전환 (D-03, D-04).

**대상 컴포넌트:**
1. `StoreBasicSection.tsx` -- 업종 2x2 grid 3곳, 규모+가격대 grid 1곳
2. `BrandVisionSection.tsx` -- grid-cols-2 없음 (수직 스택, 변경 불필요)
3. `PersonaSection.tsx` -- grid-cols-2 없음 (수직 스택, 변경 불필요)

**변경 패턴:**
```
grid grid-cols-2 gap-4  ->  grid grid-cols-1 lg:grid-cols-2 gap-4
```

## Touch Target Contract (MOBILE-03)

모바일에서의 터치 최적화 규칙.

| Element | Desktop (현행) | Mobile 최소 | 구현 방법 |
|---------|---------------|------------|----------|
| 추천 받기 버튼 | `px-5 py-2.5` (약 40px 높이) | 44px 높이 | `py-3 lg:py-2.5` |
| API설정 버튼 | `px-3 py-2` (약 32px 높이) | 44px 높이 | `py-3 lg:py-2` |
| 네이밍 초기화 버튼 | `px-3 py-2` (약 32px 높이) | 44px 높이 | `py-3 lg:py-2` |
| Dropdown select | `py-3` (약 44px 높이) | 44px 높이 | 이미 충분 (유지) |
| TextArea | `py-2.5`~`py-3` | 44px 최소 | 이미 충분 (유지) |
| MiniRecommendButton | `px-2 py-0.5` (약 24px) | 변경 불필요 | 보조 버튼, 터치 타겟 예외 |
| 그룹 헤더 토글 | `px-4 py-3` (약 40px) | 44px 높이 | `py-3.5 lg:py-3` |

## Interaction Contract

### 자동 스크롤 (D-02)

- **트리거:** 추천 버튼 클릭 -> API 응답 수신 후 (로딩 완료)
- **대상:** 추천 패널 영역 최상단 (RecommendPanel의 "추천 결과" 헤더)
- **조건:** 모바일 (< 1024px)에서만 작동. 데스크톱에서는 우측 패널이 항상 보이므로 불필요
- **구현:** `ref.scrollIntoView({ behavior: 'smooth', block: 'start' })`
- **오프셋:** 상단에서 16px 여유 (scroll-mt-4)
- **타이밍:** API 응답 후 100ms 지연 (DOM 렌더링 대기)

### 뷰포트 메타 태그

`index.html`에 반드시 포함 확인:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## Copywriting Contract

이 Phase는 새로운 UI 텍스트를 추가하지 않는다. 기존 카피를 모바일에서 그대로 사용한다.

| Element | Copy | 변경 |
|---------|------|------|
| Primary CTA | 추천 받기 | 유지 |
| Empty state heading | 추천 대기 중 | 유지 |
| Empty state body | 왼쪽 항목을 입력하고 추천 받기를 눌러보세요 | 모바일: "위의 항목을 입력하고 추천 받기를 눌러보세요" |
| Error state | {error 메시지 그대로 표시} | 유지 |
| Destructive | 네이밍 초기화 | 유지 |

**모바일 전용 카피 변경 1건:**
- EmptyState 안내 문구: "왼쪽 항목을" -> 모바일에서 "위의 항목을" (레이아웃이 세로 스택이므로)
- 구현: `<span className="lg:hidden">위의</span><span className="hidden lg:inline">왼쪽</span> 항목을 입력하고`
