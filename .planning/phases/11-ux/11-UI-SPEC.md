## Spacing Scale

Declared values (all multiples of 4, matching Phase 8 conventions):

| Token | Value | Usage in Phase 11 |
|-------|-------|-------------------|
| xs | 4px | Tab indicator thickness area, icon gaps |
| sm | 8px | TextField internal gaps, between group header icon and title |
| md | 16px | Default field-to-field gap inside a persona group (`space-y-4`) |
| lg | 20px | Card padding mobile (`p-5`) — matches existing `p-5` sections |
| xl | 28px | Card padding desktop (`lg:p-7`) — matches existing `lg:p-7` sections |
| 2xl | 32px | **Gap BETWEEN 5 persona group containers** (`space-y-8`) |
| 3xl | 44px | Tab bar height (`h-[44px]`, unchanged from Phase 8) |

**Exceptions:**
- Tab bar height `44px` is a touch-target exception (Apple HIG minimum), retained from Phase 8. Not a 4-multiple of 8 but a hard touch-target requirement.
- 5-group persona stack uses `space-y-8` (32px) — stronger than intra-section `space-y-6` to make group boundaries unambiguous without an accordion.

## Typography

Single type ramp — 3 sizes, 2 weights (matches Phase 8 contract). No new sizes introduced.

| Role | Size | Weight | Line Height | Phase 11 usage |
|------|------|--------|-------------|----------------|
| Body / Field value | 14px | 400 regular | 1.5 | TextField/TextArea input text, 관점 helper 문구 |
| Label / Group header title | 14px | 600 semibold | 1.4 | SectionHeader h2, 필드 라벨 |
| Subheading / Group eyebrow | 12px | 500 medium | 1.4 | 그룹 헤더 상단 eyebrow ("그룹 1 · 브랜드 정체성"), 관점 배지 텍스트 |
| Tab label | 14px | 600 semibold | 1.0 | WizardTabs 버튼 |

**Hierarchy rule for persona tab:**
- Group header (SectionHeader h2) = **14px semibold** ← same level as any other section on other tabs. The group is a section, not a sub-section.
- Field label (inside TextField) = existing 12–13px regular (untouched).
- No "page title" typography — the tab itself is the page-level context.

**What changes for Phase 11:** SectionHeader gains an optional `subtitle` prop slot for the 12px eyebrow text used by persona group cards (e.g. "1 / 5 · 브랜드 정체성"). Implementation is additive; existing call sites unaffected.

## Color

60 / 30 / 10 distribution is preserved from v1.x; Phase 11 adds **zero new hues**.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#2C2825` | Page bg, sticky tab bar bg, InputPanel container |
| Dominant ink on dark | `#D0CAC2` / `#A09890` | Inactive tab text, placeholder copy on dark |
| Secondary (30%) | `#E8E4DE` | All 5 persona group cards, all section cards across tabs (MarketTrend, BrandVision, Competitor, etc.) |
| Secondary border | `#C5BFB7` | Group card border |
| Secondary ink | `#2C2825` | SectionHeader h2, field labels, field values |
| Secondary muted ink | `#6B6560` | 12px eyebrow, 관점 helper text, placeholders |
| Accent (10%) | `#B48C50` | **Reserved list below — strictly enforced** |
| Destructive | not used in Phase 11 | no destructive actions introduced |

**Accent `#B48C50` reserved for — explicit list (no others may use gold):**
1. Active tab text (`WizardTabs` active state)
2. Active tab sliding indicator (`h-[2px]` bar under current tab)
3. SectionHeader accent bar (`w-1 h-4 rounded-full bg-[#B48C50]`) — left of each group title
4. `RecommendButton` primary CTA background
5. `MiniRecommendButton` icon hover state
6. `KeywordWeightSlider` thumb and active track

**Persona group differentiation rule (D-05 Claude discretion):**
All 5 group containers use the **identical** card style (`rounded-2xl bg-[#E8E4DE] p-5 lg:p-7 border border-[#C5BFB7]`). **No per-group hue variation.** Differentiation is achieved via:
- Numbered eyebrow: "1 / 5 · 브랜드 정체성"
- Unique lucide icon per group (see Copywriting section)
- 32px vertical gap between groups (`space-y-8`) — stronger than default
- Section title text

This preserves the "심플 + 고급스러움" constraint from CLAUDE.md. Per-group color tints would fragment the palette and violate the 10% accent rule.

**관점(Perspective) badge color:**
- 분석 탭 badges ("시장 현황 관점") = `#6B6560` text on `#DDD7CF` chip background — neutral, informational.
- 페르소나 탭 badges ("브랜드 내부 관점") = `#6B6560` text on `#DDD7CF` chip background — identical treatment so neither feels "more important."
- Both use the same neutral chip. Semantic difference is in the copy, not the color.

## Layout

### 4-tab width math (D-02)

- `WizardTabs` buttons: `flex-1` (automatic equal split). No code change needed for button sizing.
- Sliding indicator: `width: ${100 / 4}%` (was `100 / 3`). `transform: translateX(${activeIndex * 100}%)` unchanged — still moves by full indicator-width steps.
- Tab bar height: `h-[44px]` unchanged.
- Tab label size: `text-[14px]` unchanged.
- Tab bar sticky wrapper: `sticky top-0 z-10 bg-[#2C2825] -mx-5 lg:-mx-8` (already in InputPanel — unchanged).

### 4-tab mobile responsive (Claude discretion)

Korean labels are 2 chars (분석, 정체성, 페르소나, 표현) = max 3 chars. At 360px viewport width (smallest supported), each tab = 90px, well above touch target minimum. **Decision: no horizontal scroll, no shrinking, no icon-only fallback.** Keep flex-1 equal split across all breakpoints. This is the simplest option and the labels fit comfortably.

If future locales add longer labels, revisit. For Phase 11: static 4-way split on all viewports.

## Interaction

### Tab switching
- Click/tap any tab → `onTabChange(id)` → updates Zustand `activeTab`.
- Sliding indicator transitions 300ms ease-in-out (existing — unchanged).
- Content area does NOT animate (existing) — immediate swap via conditional render.
- No unsaved-state warning (fields auto-persist to Zustand on every keystroke, no data loss across tab switches).

### Keyboard navigation
- Tab bar: `role="tab"` already set. Add `role="tablist"` to parent `<div>` (currently missing — required for correct SR semantics with 4 tabs). Add `aria-selected` — already present.
- Left/Right arrow keys cycle through tabs when focus is inside tab bar. **New in Phase 11:** implement arrow key handling (doesn't exist in current WizardTabs.tsx). Arrow Left → previous tab, Arrow Right → next tab, wraps at boundaries.
- Home → first tab, End → last tab.
- Tab key: moves focus into the active tab's panel content (natural browser order).

### Focus states
- Tab buttons: add visible focus ring `focus-visible:outline-2 focus-visible:outline-[#B48C50] focus-visible:outline-offset-[-2px]` (currently missing — 6 dimension checker will fail without it).
- Field focus rings: existing TextField behavior unchanged.

### RecommendButton / KeywordWeightSlider placement (Claude discretion, locked here)
- PersonaTab follows the established Phase 8 pattern: slider + button at the **bottom of the tab**, after group 5. Not inside any group.
- Mini-recommend buttons on individual fields: **remove for Phase 11 persona tab.** The original PersonaSection wrapped each of 16 fields with `<MiniRecommendButton />`. With 16 fields distributed across 5 cards, 16 mini-buttons create visual noise. Persona tab relies on the single bottom-CTA pattern consistent with other tabs. If users want per-field AI assist, it returns in a later phase as a right-aligned icon with distinct treatment.
- Other tabs (analysis/identity/expression): no change to existing mini-button presence.

### 관점 badge interaction
- Static chip. Not clickable. No tooltip in Phase 11 (deferred — a tooltip with the sister-tab reference is a nice-to-have but out of scope for this phase).

### Scroll behavior
- Persona tab will exceed viewport. Sticky tab bar remains at top while user scrolls through 5 groups. No scroll-spy, no "jump to group" nav — the 5-group stack is short enough (est. ~1800px total on desktop) that manual scroll is fine.

### Animations
- No new animations introduced. Existing `animate-fadeIn` (Phase 8, index.css @theme) remains for RationaleExpandedCard and is unrelated to this phase.
